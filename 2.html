2.2 División del conjunto de datos¶

2.2.1. Hold-out¶

Consiste en dividir el conjunto de datos en dos, una para entrenar el modelo y otra para testearlo. Tambin se recurre al hold-out en caso de que el modelo a utilizar tenga parmetros que ajustar. Para ello la particin de entrenamiento se divide a su vez en entrenamiento y validacin. El conjunto de entrenamiento se usa para entrenar el modelo probando diferentes combinaciones de parmetros, y el conjunto de validacin para comprobar el funcionamiento del modelo con dichas combinaciones. Al final se escoge el conjunto de parmetros que mejor resultados haya obtenido, y como paso final se reentrena el modelo utilizando todo el conjunto de entrenamiento. En la siguiente prctica usaremos los mtodos vistos en sta para ajustar los parmetros de un modelo de regresin lineal.

Es importante resaltar que en todo este proceso no se usa para nada el conjunto de test, cuya finalidad es nicamente comprobar el rendimiento del modelo final.

En la prctica anterior vimos como scikit-learn nos ofrece una funcin para realizar este tipo de divisin del conjunto de datos, train_test_split. En esta prctica vamos a ver cmo podemos hacerlo "a mano"!.

$R^2$ es una buena mtrica, pero la mayora de las veces queremos validar nuestro resultados usando varias (no nos fiamos de una sola, por qu?)

Mtricas muy usadas son el root mean square error (RMSE) o el mean absolute error (MAE)

  • RMSE = $\sqrt{\frac{1}{N}\sum_{i=0}^{N-1}(y_i-\hat{y_i})^2}$
  • MAE = $\frac{1}{N}\sum_{i=0}^{N-1}|y_i-\hat{y_i}|$
In [37]:
def rmse(y, yp):
    return np.sqrt(np.sum((y-yp)**2) / y.shape[0])

Fíjate en dos cosas:¶

  2. Toma nota de estos valores para compararlos con los que obtengamos despus.

El obtener resultados usando un mismo conjunto para todo (entrenamiento y test) nos puede dar (y seguramente lo har) una versin poco realista del rendimiento del modelo. Otro riesgo adicional es que el modelo puede "sobreentrenarse", aprendindose los datos de entrenamiento "de memoria" pero siendo incapaz de predecir correctamente sobre datos que no haya "visto" previamente. A este fenmeno se le conoce como overfitting y hay que tratar de evitarlo al entrenar nuestros modelos.

2.2.2 k-fold¶

Una pega del mtodo hold-out es que al particionar aleatoriamente el conjunto de datos podra darse el caso que justamente esta particin fuera especialmente idnea a la hora de entrenar y testear, dndonos resultados aparentemente demasiado buenos.

Una opcin para evitar esta duda es particionar el conjunto varias veces e ir probando diferentes combinaciones de conjuntos de entrenamiento y test. Si esto lo hacemos de manera sistemtica tenemos el k-fold (o v-fold), ilustrado en la figura.

En esta ocasin vamos a utilizar las funciones de scikit-learn para realizar las particiones

Ejercicio. Completa el cdigo mostrado abajo para entrenar y testear con k-fold.

%%time
# Usamos KFold de scikit-learn
from sklearn.model_selection import KFold

nf = 5  # number of folds
kf_r2, kf_rmse, kf_mae = [], [], []  # Listas para ir almacenando los resultados
kf = KFold(n_splits=nf, shuffle=True, random_state=1234)
for train, test in kf.split(Xtrain):  # pregunta: dara igual poner kf.split(X,y)? si, obtenemos el mismo resultado
    XtrainF, ytrainF = Xtrain[train, :], ytrain[train]
    # Completa -----------------------
    XtestF, ytestF = Xtrain[test, :], ytrain[test]

    model = LinearRegression()
    model.fit(XtrainF, ytrainF)

    yp = model.predict(XtestF)
    r2 = model.score(XtestF, ytestF)
    rmse = np.sqrt(mean_squared_error(ytestF, yp))
    mae = mean_absolute_error(ytestF, yp)

    kf_r2.append(r2)
    kf_rmse.append(rmse)
    kf_mae.append(mae)

2.2.3. leave-one-out¶

El leave-one-out es la versin extrema del k-fold con k = N muestras. Este es el test ms exhaustivo de todos los vistos, y el que tericamente mejor podra predecir el "verdadero" error de prediccin de un modelo. Dado su potencialmente alto coste computacional (hay que entrenar N modelos con N-1 muestras) se usa solo cuando tenemos pocos datos.

%%time
# Pasar usar leave-one-out basta con usar KFold con N muestras.
# No obstante, scikit-learn provee the una clase LeaveOnOut directamente.
# Importante
from sklearn.model_selection import LeaveOneOut
loo = LeaveOneOut()

# Pregunta: para este experimento no usaremos el R2, por qu? NO podemos 
# hacer un ajuste lineal con solo un punto 
# y por tanto no se puede calcular la R2 del ajuste...
loo_rmse, loo_mae = [], []
for train, test in loo.split(X):
    Xtrain, ytrain = X[train, :], y[train]
# Completa ------------------------
    Xtest, ytest = X[test, :], y[test]
    
    model = LinearRegression()
    model.fit(Xtrain, ytrain)

    yp = model.predict(Xtest)

    rmse = np.sqrt(mean_squared_error(ytest, yp))
    mae = mean_absolute_error(ytest, yp)

    loo_rmse.append(rmse)
    loo_mae.append(mae)
    
        

print('RMSE: ', np.mean(loo_rmse))
print('MAE: ', np.mean(loo_mae))

ax2.set_xlabel('Particin')
ax2.set_ylabel('MAE')

plt.tight_layout()
plt.show()

El modelo se entrena con la mayora de los datos en cada particin y solo se excluye un punto de datos a la vez, lo que conduce a predicciones muy similares en todas las particiones, esto hace que haya similitud en las grficas de RMSE y MAE.

LeavePOut: es como KFold pero al revs, se escogen P muestras del conjunto de test. Ojo porque esto no es equivalente a hacer KFold(n_splits=n_samples // p). LeavePOut prueba todas las combinaciones posibles de P elementos para testear, dejando las restantes N-P muestras para entrenar. Esto tiene un muy alto coste computacional si la combinatoria de N elementos tomados en grupos de P es alta (y es fcil que lo sea).

from sklearn.model_selection import LeavePOut
